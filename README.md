# 上海火锅商户数据分析（MySQL + Python）

基于大众点评749家上海火锅店数据，分别用Python和MySQL做分析，练了一遍从数据清洗到聚类建模的流程。

## 项目结构（STAR）

### 背景（Situation）
课程练习项目。拿到一份上海火锅商户数据，想练一下完整的数据分析流程：清洗→统计→可视化→建模→输出。

### 任务（Task）
- 用MySQL做数据清洗和基础查询
- 用Python做可视化、相关性分析、聚类和词云
- 看看数据里能发现什么规律

### 行动（Action）

**MySQL部分**
- 建表导入749条商户数据
- 修正价格异常值（1元→111元）
- 评论数空值用中位数填充
- 新增数字星级列（五星→5.0，准五星→4.5，依此类推）

**Python部分**
- 画价格分布、星级分布、评分箱线图
- 算相关系数，画热力图
- 推荐菜品做Jieba分词，生成词云
- K-Means聚类（选星级/口味/环境/服务四个维度，K=3）
- Apriori关联规则挖掘菜品组合
- 写了一个按价格区间和关键词筛选店铺的函数

### 结果（Result）

**价格分布**
- 100-200元区间店铺最多，评论数也最高，占总评论量35.7%

**相关性**
- 环境评分与价格相关系数0.74
- 口味评分与价格相关系数0.57
- 星级与口味相关性最强

**聚类结果（K=3）**
| 类别 | 数量 | 人均 | 口味均分 |
|------|------|------|----------|
| 高端精品 | 21家 | 216元 | 8.9 |
| 大众口碑 | 412家 | 143元 | 8.0 |
| 低价基础 | 316家 | 68元 | 7.2 |

**关联规则**
- 毛肚→鹅肠，置信度0.82
- 牛肉→虾滑，置信度0.76

**高频推荐菜Top5**
毛肚(89) > 牛肉(76) > 虾滑(62) > 鹅肠(51) > 黄喉(43)
